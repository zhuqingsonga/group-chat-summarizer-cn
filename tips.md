# 群聊总结 - 使用技巧

## 一句话触发指南

最自然的触发方式，直接说：

```
"总结一下"
"群聊总结"
"整理一下记录"
"今天群里说了啥"
"提取待办"
"分析氛围"
```

## 快速开始

### 1. 复制群聊记录

**飞书**：
1. 打开群聊 → 右上角"..." → "聊天记录"
2. 选择时间范围 → "导出"
3. 复制文本内容

**钉钉**：
1. 群设置 → "聊天记录"
2. 选择日期 → "导出"

**Discord**：
1. 使用 DiscordChatExporter 导出 JSON
2. 或手动复制相关消息

### 2. 粘贴并总结

```
你：总结一下这段对话
[粘贴聊天记录]

SiVi：生成结构化总结...
```

## 提高准确率的技巧

### 提供上下文

❌ 不好：
```
总结一下
[1000 条消息]
```

✅ 更好：
```
这是产品技术群昨天的讨论，请生成详细总结
[1000 条消息]
```

### 指定重点

```
提取这段对话的所有待办事项
```

```
分析这个讨论的情感氛围，有没有争议
```

```
只输出确定了什么、还有什么没决定
```

## 常见问题

**Q: 为什么有些 @人名没识别出来？**

A: 检查原始记录中的 @格式。有些平台导出时会丢失 @信息。

**Q: 截止时间识别不准？**

A: 相对时间（"下周"、"周五"）基于消息时间计算。如果消息时间戳缺失，可能无法推断。

**Q: 可以处理图片/语音吗？**

A: 目前只能处理文本。图片中的文字需要 OCR 后粘贴。

**Q: 支持多语言混合吗？**

A: 支持中英混合，但情感分析对中文更准确。

## 输出格式自定义

默认输出 Markdown 表格，你可以要求：

```
以纯文本列表形式输出待办
```

```
用 JSON 格式输出，方便我导入其他工具
```

```
只输出可以复制到 Excel 的表格
```

## 隐私提醒

⚠️ **敏感信息处理**：
- 群聊记录可能包含敏感信息
- 分享总结前请脱敏处理
- 不建议在公共 AI 服务处理机密对话

## 批量处理

如需定期总结（如每日站会）：

1. 设置定时导出群聊记录
2. 使用脚本调用 summarize_chat.py
3. 自动发送到指定邮箱/群

示例命令：
```bash
python scripts/summarize_chat.py chat_log.txt --level standard --output summary.md
```

## 触发优化建议

### 让 AI 更容易理解你的意图

**❌ 模糊的表达：**
```
"看看这个"
"处理一下"
"分析一下"
```

**✅ 清晰的触发：**
```
"总结这个群聊记录"
"提取这段对话的待办事项"
"分析这个讨论的情感氛围"
```

### 上下文关键词

在请求中加入这些词，触发率更高：

| 关键词类型 | 示例词汇 |
|-----------|---------|
| 动作词 | 总结、整理、提取、分析、生成 |
| 对象词 | 群聊、对话、记录、讨论、会议 |
| 目标词 | 待办、纪要、报告、摘要、同步 |
| 场景词 | 今天、昨天、这周、刚才、错过 |

### 常见表达方式

**自然口语：**
- "我这两天不在，群里说了什么？"
- "刚才讨论太长了，帮我整理一下"
- "把今天的讨论总结一下发给老板"

**工作场景：**
- "生成今日站会纪要"
- "提取项目群本周待办"
- "分析客户反馈群的情绪"

**紧急场景：**
- "快速总结一下，我要汇报"
- "错过群聊了，给我个摘要"
- "刚才争论什么，谁要做什么"
