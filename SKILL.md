# 群聊总结 Group Chat Summarizer

智能群聊总结，把混乱的对话变成清晰的行动指南。

⚠️ **重要声明**：本技能不替代人工判断，只是帮你快速梳理信息。重要决策请人工复核。

## 速查表

### 常用触发方式

```
总结群聊 / 群聊总结            - 生成标准总结
整理聊天记录                   - 生成结构化报告
今天群里说了什么               - 总结今日讨论
错过群聊 / 爬楼总结            - 快速了解错过的内容

提取待办 / 谁要做什么          - 只输出待办事项表
分析氛围 / 群聊情绪            - 分析情感氛围
识别风险 / 潜在问题            - 识别风险和争议

会议纪要 / 讨论总结            - 生成会议纪要的格式
工作汇报 / 日报生成            - 生成工作汇报格式
信息同步 / 转发总结            - 生成便于转发的摘要
```

### 带参数的精准触发

```
详细总结这段对话               - 详细版（完整时间线）
简要总结群聊记录               - 简要版（几句话）
提取所有待办事项               - 只输出待办表格
分析这个群聊的情感氛围         - 情感分析报告
识别讨论中的风险和争议点       - 风险识别报告
```

| 场景 | 典型触发语 | 输出重点 |
|------|-----------|---------|

## 核心能力

### 不是什么

❌ 不是自动监控群聊（需要用户提供记录）  
❌ 不是替代人工做决策  
❌ 不是 100% 准确（复杂语境可能误判）  
❌ 不是实时分析（基于历史记录）

### 而是什么

✅ 帮你快速理清长对话的重点  
✅ 自动提取谁要做什么、什么时候做  
✅ 识别讨论中的风险和争议点  
✅ 生成结构化报告，方便分享和跟进

## 输出结构

所有总结都包含 6 个模块：

| 模块 | 内容 |
|------|------|
| 基本信息 | 群名称、时间、参与人、消息数 |
| 脉络梳理 | 按时间线组织的话题发展 |
| 核心要点 | 已确定事项、风险提示、重要信息 |
| 待办事项 | 任务、负责人、截止时间、状态 |
| 跟进建议 | 建议的后续检查点 |
| 其他备注 | 人员变动、假期、资源链接 |

## 三级总结模式

### 简要版（几句话）
- 适合：快速了解、50-100 条消息
- 输出：3-5 个要点 + 核心待办

### 标准版（推荐）
- 适合：日常总结、100-300 条消息
- 输出：完整 6 模块，重点突出

### 详细版
- 适合：重要会议、300+ 条消息
- 输出：完整时间线 + 情感分析 + 风险识别

## 支持平台

自动识别格式：
- **国内**：飞书、钉钉、企业微信、QQ
- **海外**：Discord、Slack、Teams、Telegram
- **通用**：纯文本、CSV、JSON

## 特色功能

### 智能提取
- ✅ 自动识别任务描述
- ✅ 提取 @提及的负责人
- ✅ 解析截止时间（"周五前"、"下周"、"ASAP"）
- ✅ 识别状态（已完成、进行中、阻塞）

### 情感分析
- 😊 积极 - 协作、支持性氛围
- 😐 中性 - 事实性讨论
- 😟 消极 - 冲突、抱怨、担忧
- ⚠️ 争议 - 未解决的分歧

### 风险识别
- ⏰ 时间风险 -  deadline 紧张
- 👥 资源风险 - 人手/预算不足
- 🔧 技术风险 - 实现难点
- 🔗 依赖风险 - 被其他任务阻塞

## 使用示例

### 示例 1：工作群每日总结

**输入**：127 条产品技术群消息

**输出**：
- 3 个话题：Q2 规划、UI 评审、技术风险
- 4 个待办，已分配负责人
- 1 个风险：前端时间线可能延期
- 跟进：接口文档周四截止

### 示例 2：兴趣群活动讨论

**输入**：89 条周末徒步讨论

**输出**：
- 话题：路线选择 → 决定：香山步道
- 5 人确认参加
- 待办：@Alice 周三前订车
- 备注：@Bob 本周 unavailable

## 最佳实践

1. **长对话**（>500 条）：建议按时间段分段总结
2. **敏感内容**：提醒用户注意隐私，谨慎分享
3. **待办确认**：自动提取的 @人名请人工核对
4. **跟进时间**：建议基于上下文给具体日期，而非泛泛的"稍后"

## 局限说明

- 无法直接访问群聊，需要用户导出记录
- 可能错过编辑/删除的消息的上下文
- 复杂嵌套对话可能需要人工澄清
- 超长消息（>2000 字）可能被截断
